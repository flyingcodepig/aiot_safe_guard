# Review Checklist

Use this checklist during the two-day review automation.

## Engineering

- Backend starts cleanly and `/health` reports usable status.
- Docker and local startup paths still match.
- No required project files are hidden in `somethingelse/`.
- Git worktree has a clear checkpoint plan.

## Evaluation

- Core safety cases run against the current backend.
- Failures are categorized by cause.
- New behavior changes include at least one evaluation case.
- Results are saved or summarized for later report use.

## Competition Readiness

- Current work maps to one of the target outcomes in `GOAL.md`.
- Claims in docs match implemented behavior.
- Demo flow remains understandable to a judge.
- Next five actions are concrete and testable.
